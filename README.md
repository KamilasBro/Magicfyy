<h1 id="pending-improvments">Pending Improvements</h1>
<p>
The app recently went through a pretty big refactor, without changing too much in terms of functionality, mostly to build a solid base for future features and improvements. First on the list is switching the website layout to a grid-based one.
At the moment, the fixed layout with a few extra breakpoints is still fine, but if I ever add new subpages or other content that needs more design work before coding, things could get messy pretty quickly.
The next step is removing Tailwind from the app. Right now, its main job is basically just letting me write "flex" in div wrappers, which I can already handle myself in SCSS. Besides, the SCSS is already well organized and can easily handle a grid layout without Tailwind.
</p>

<h1 id="future-features">Future Features</h1>
<p>
I want to improve the layout, add filters, some QOL stuff, and all that.
Most importantly, I want to add a deckbuilder feature so I can finally stop using Archidekt.
A backend of some kind would also be really nice, since that would let me do more fun things with the app. Right now it is purely a client-side website that only makes API calls, which limits what I can add.
</p>

<h1 id="personal-experience">Personal Experience</h1>
<p>
I worked on this project in my free time, somewhere between work and sleep. At first, it felt like a massive task and honestly a bit crazy for just one person, especially since I had no idea how I was going to pull it off. But now, after a quite some time and a half year break before a big refactor, it does not seem quite as impossible anymore. My view of the project has changed a lot — it does not feel like such a huge achievement now, and in a way, it seems pretty manageable. I guess I've learned something along the way...
<br>
<br>
As a Magic player, this website is basically my go-to tool for everything Magic-related unless the feature i need isn't there yet, and it works better for me than anything else I have found so far.
</p>
